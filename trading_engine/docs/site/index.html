Overview

Bubbles is a self-hostable algorithmic trading sandbox built around a price–time-priority matching engine written in C++. Bots connect over WebSocket, place orders that match against a built-in market maker or each other, and stream the live order book, trades, and fills.

The engine exposes two interfaces: a REST API for market-data snapshots, bot management, and health, and a WebSocket API for real-time streaming and order entry. Examples below assume the engine is running on localhost:9090.

All timestamps are Unix epoch milliseconds. Prices are double, quantities and IDs are unsigned integers. Sides are "Buy"/"Sell" and are case-insensitive on input.

Markets

Each market's top of book is anchored to a real-world reference price; the market maker fabricates the depth ladder beneath it. Index anchors come from Yahoo Finance (real-time); ETF anchors come from Massive (~15-min delayed).

Authentication

Order entry and bot management require a per-user API key. Issue one from the dashboard; the engine validates it against the backend and caches the result.

PropertyValue
Format^sk_live_[a-f0-9]{32}$
Examplesk_live_a1b2c3d4e5f6a7b8c9d0e1f2a3b4c5d6

A key may be supplied three ways (checked in order):

Authorization: Bearer sk_live_...
Api-Key: sk_live_...
?api_key=sk_live_...        # query parameter
Market data (order book, trades, health) is public. Only order entry, the WebSocket upgrade, and bot pause/resume require a key.

REST · Trades

GET/trades?symbol={sym}&limit={n}public

Most recent executions. limit defaults to 50 (max 500). Omit symbol for all markets.

{
  "trades": [
    { "trade_id": 10001, "symbol": "NIKKEI", "price": 66021,
      "quantity": 1, "taker_side": "Buy", "ts": 1781276135672 }
  ]
}
GET/trades/historical?symbol={sym}&from={ms}&to={ms}&limit={n}public

Time-ranged history from the persistent store. limit defaults to 5000 (max 50000). Returns a count alongside trades.

Cancels the bot's resting orders and disconnects it. Owner only; internal bots can't be paused.

WebSocket · Connecting

WSws://localhost:9090/ws?api_key=sk_live_...auth

Upgrade with the API key in the query string (or an Authorization/Api-Key header). On success the server immediately sends a welcome frame.

All frames are text (UTF-8 JSON) with a required "t" type field. Two streaming channels are available: book and trades.

WebSocket · Client → Server

hello identify

Attach a label so your orders are attributed in the dashboard.

{ "t": "hello", "client_id": "my-bot" }

subscribe market data

Channel is "book" or "trades". A book subscription replies with a full snapshot, then streams book_delta frames.

{ "t": "subscribe", "channel": "book", "symbol": "NIKKEI", "depth": 10 }

unsubscribe

{ "t": "unsubscribe", "channel": "book", "symbol": "NIKKEI" }

place_order auth

limit_price is required for Limit orders. client_order_id is optional and echoed back.

  "client_order_id": "o-1",
  "symbol": "NIKKEI",
  "side": "Buy",
  "type": "Limit",
  "quantity": 5,
  "limit_price": 66020
}

cancel_order

{ "t": "cancel_order", "order_id": 12345 }

ping

{ "t": "ping" }

WebSocket · Server → Client

welcome

{ "t": "welcome", "user_id": "user_123", "server_time": 1781276135672 }

book snapshot

Levels are [price, qty] pairs. seq is monotonic per symbol — track it to detect gaps.

{
  "t": "book", "symbol": "NIKKEI", "snapshot": true,
  "seq": 1000, "ts": 1781276135672,
  "bids": [[66020, 9], [66019, 16]],
  "asks": [[66021, 7], [66022, 16]]
}

book_delta incremental

Apply changes in order. qty = 0 deletes a level; qty > 0 sets it. If seq is not exactly previous + 1, re-subscribe to resync.

{
  "t": "book_delta", "symbol": "NIKKEI",
  "seq": 1001, "ts": 1781276135890,
  "bids": [[66020, 12], [66015, 0]],
  "asks": [[66021, 0]]
}

WebSocket · Example session

// 1. connect
ws = new WebSocket("ws://localhost:9090/ws?api_key=sk_live_...")

// 2. server greets
<- { "t": "welcome", "user_id": "user_123", "server_time": 1781276135672 }

// 3. identify + subscribe
-> { "t": "hello", "client_id": "my-bot" }
-> { "t": "subscribe", "channel": "book", "symbol": "NIKKEI" }
<- { "t": "book", "symbol": "NIKKEI", "snapshot": true, "seq": 1000, ... }
<- { "t": "book_delta", "symbol": "NIKKEI", "seq": 1001, ... }

// 4. trade
-> { "t": "place_order", "symbol": "NIKKEI", "side": "Buy",
      "type": "Limit", "quantity": 5, "limit_price": 66021 }
<- { "t": "order_ack",  "order_id": 12345, "status": "Pending", ... }
<- { "t": "order_fill", "order_id": 12345, "status": "Filled", ... }
